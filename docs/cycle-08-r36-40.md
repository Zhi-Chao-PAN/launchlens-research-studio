# Rounds 36-40 — Security Hardening Cycle

Five rounds focused on hardening the API security surface, adding
observability, and stabilizing visual regression testing.

## What was built

### Authentication & Access Control (R36, R37)
- **Token scope system**: two-tier tokens — `bypass` (skips rate limits
  on /api/research) and `admin` (full access to /api/admin/* endpoints
  + bypass). Bypass-scope tokens are rejected from admin routes.
- **Admin rate limiting**: /api/admin/* endpoints get their own token
  buckets — 30 req/min per IP, 100 req/min per token. The stricter of
  the two applies. Prevents a leaked admin token from being used to
  create unlimited bypass tokens.
- **Env token provisioning**: `LAUNCHLENS_BYPASS_TOKENS` and
  `LAUNCHLENS_ADMIN_TOKENS` env vars for initial token bootstrapping.
- **Token metadata**: every token now tracks `lastUsedAt`, `lastIp`,
  `usageCount`, `scope`, and `label`.

### Audit Log (R37)
- **`src/lib/api/auth-audit.ts`**: 100-event ring buffer of security
  events. Persisted via the storage backend.
- **7 event types**: `token_created`, `token_revoked`, `auth_success`,
  `auth_failed`, `csrf_failed`, `rate_limited`, `admin_action`.
- **Structured fields**: ipHash, tokenHash, scope, detail, userAgent.
- **Admin endpoint**: `GET /api/admin/audit?limit=N` (admin scope only).

### End-to-End Security Tests (R38)
- **`e2e/security-e2e.js`** — 10 tests: CSRF token issuance, soft
  enforcement, mismatched token 403, rate-limit activation, admin
  endpoint auth rejection (no auth, fake token, audit endpoint).
- **`e2e/admin-e2e.js`** — 21 tests: env-provisioned admin token,
  token CRUD, scope isolation, bypass token actually skips rate limits,
  audit log queries, limit parameter, token revocation, admin
  rate-limiting.
- **Total: 31 security e2e tests**, all passing, runs in ~4s.

### Visual Regression Stabilization (R39)
- Fixed pre-existing `gotoUrl is not defined` crash in mobile e2e.
- Replaced flaky "Research Agents" heading selector with
  `button[aria-controls="studio-sidebar"]` as the studio-layout signal.
- Added `settle(page, ms)` helper: scroll-to-top + CSS animation
  suppression + configurable wait, applied to all 8 screenshot points.
- Removed duplicate security tests from the browser e2e suite
  (covered by dedicated test files).
- **Result: 8/8 pixel-perfect match in freeze mode (0 drift)** — down
  from 5 mismatches + 2 missing baselines.

### Regression Suite (R38, R40)
- Regression script now runs: lint → unit tests → build → bundle stats
  → Browser E2E → Security E2E → Admin E2E.
- Reordered so browser E2E runs before API E2E (avoids rate-limit
  starvation).
- `PLAYWRIGHT_SKIP=1` to skip all e2e for fast local iterations.
