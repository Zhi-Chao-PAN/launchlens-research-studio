# 模型评估辅助系统 · 产品需求文档 (PRD)

> 版本: v0.1 · 最后更新: 2026-06-17
> 项目代号: ModelEval Studio（暂定名）

---

## 1. 项目背景

我们经常参加某大厂的模型测试，需要对多个匿名模型（如 sigma/quartz/raven/prism）的表现进行评估和反馈。当前流程依赖人工逐个分析、填写评分，效率低且标准不统一。

本系统旨在提供一个 AI 辅助的模型评估工作台，帮助测试人员基于个人背景、任务描述和模型产物，快速生成专业的评估报告。

---

## 2. 核心目标

- **提升效率**: AI 辅助生成评估内容，减少人工撰写时间
- **统一标准**: 基于用户背景和任务上下文，给出结构化、专业的评估
- **多模态输入**: 支持任务看板截图自动识别、产物上传解析
- **协作友好**: 邀请码机制，管理员可管理用户和任务

---

## 3. 用户角色

### 3.1 普通用户（测试人员）
- 使用邀请码注册账号
- 设置个人背景资料
- 创建和管理测试任务
- 使用 AI 辅助分析模型表现
- 生成和导出评估报告

### 3.2 管理员（1 人）
- 生成/禁用邀请码
- 查看用户列表和活跃情况
- 查看用户的任务记录
- 重置用户密码/背景
- 管理员独立账号密码登录

---

## 4. 核心工作流

```
邀请码登录 → 设置用户名 → 设置个人背景
    ↓
创建测试任务（场景/需求类别/任务说明/附件）
    ↓
AI 基于背景 + 任务 → 生成测试思路（可对话调整）
    ↓
上传任务执行看板截图（多模态识别模型列表/状态/指标）
    ↓
上传各模型产物（分栏，支持粘贴/上传/zip）
    ↓
AI 基于背景 + 任务 + 看板 + 产物 → 生成评估报告（可对话调整）
    ↓
复制评估报告文本 → 粘贴到原系统提交
```

---

## 5. 功能详细需求

### 5.1 认证与用户系统

- **邀请码注册**: 输入邀请码 → 设置用户名 → 设置密码 → 完成注册
- **登录**: 用户名 + 密码登录
- **个人背景**:
  - 一段自由文本，描述用户的身份、偏好、回答风格等
  - 例如: "我是程序员，喜欢简洁回答" 或 "我是顶级人力资源专家，喜欢详细回答"
  - 首次设置后默认为该背景
  - 提供按钮随时调整
  - 每次任务可选择性临时调整

### 5.2 AI 模型接入

- 用户自行填写 API Key + Base URL
- 支持 OpenAI 兼容格式
- 自动检测 API Key 是否可用
- 自动检测模型是否支持多模态（视觉）
  - 不支持时提示: "你的模型不支持图片，请更换多模态模型"
- 用户可随时切换不同模型

### 5.3 测试任务管理

- **创建任务**:
  - 场景（文本输入）
  - 需求类别（文本/选择）
  - 任务说明（富文本）
  - 上传附件（支持 PDF/Word/PPT/ZIP 等）
- **任务列表**: 查看历史任务，支持删除和导出
- **每个任务是独立会话**，不跨任务污染上下文

### 5.4 测试思路生成

- AI 基于用户背景 + 任务信息 → 生成测试思路
- 对话式交互: 用户不满意可以继续沟通调整
- 测试思路作为后续分析的基础框架

### 5.5 看板截图识别

- 用户上传任务执行看板截图
- AI 通过多模态能力自动识别:
  - 有哪些待测模型（代号如 sigma/quartz/raven/prism）
  - 各模型的状态（已完成/进行中/失败）
  - 各项指标数据（耗时、工具调用次数等）
- 模型数量不固定（可能 3-5 个），需自动适配
- 识别结果可编辑确认

### 5.6 产物上传与分析

- **分栏展示**: 每个待测模型一栏
- **每栏支持**:
  - 文本粘贴框
  - 文件上传区
  - 支持 ZIP 包上传（需解压解析内容）
- **文档解析**:
  - PDF/Word/PPT 等二进制文档，服务端解析提取文字
  - 使用 npm 包实现

### 5.7 AI 评估分析

- **混合式交互**:
  - 结构化向导为主，流程固定
  - 每一步的 AI 输出，用户都可以对话调整
  - 类似 ChatGPT 的对话界面，但始终带着任务上下文和用户背景
- **评估报告输出格式**（文本，方便复制）:

```
产物效果反馈：
[对产物是否符合预期，以及对出现的问题进行反馈]

模型的综合表现怎么样：
评分：X.X / 10
评论：[结合产物和轨迹及交付效率等各因素的综合评论，详细专业，以用户口吻]

模型交付效率是否符合预期？
评分：X.X / 10
评论：[评价交付效率，考虑速度等效率因素]

模型的产物质量怎么样
评分：X.X / 10
评论：[评价产物质量]
```

- 每个模型一份报告
- 评分支持 0.5 粒度（如 6.5、7.5）
- 以用户口吻撰写，方便直接复制粘贴

### 5.8 数据存储

- **服务端数据库**: Neon Postgres
  - 用户信息、邀请码、任务元数据
- **用户的测试记录、截图、评分** — 存储在数据库 + 对象存储
- 暂不考虑历史趋势分析
- 支持任务删除和导出
- 不支持任务复制/模板（每次重新填）

### 5.9 管理员后台

- 邀请码管理: 生成、禁用、查看使用情况
- 用户管理: 查看用户列表、活跃情况、任务记录
- 用户操作: 重置密码、重置背景
- 管理员独立账号密码登录

## 6. 非功能需求

### 6.1 UI 风格
- 浅色专业风
- 响应式设计
- 中文版（默认中文）

### 6.2 技术栈
- 前端: Next.js 16 + React 19 + TypeScript
- 样式: Tailwind CSS v4
- 数据库: Neon Postgres
- 部署: Vercel
- AI 接入: OpenAI 兼容 API

### 6.3 安全
- 邀请码机制控制注册
- 密码哈希存储
- API Key 加密存储
- 用户数据隔离

## 8. 数据模型（初步）

### users
- id, username, password_hash, background_text, created_at, last_active_at

### invite_codes
- id, code, created_by, used_by, is_active, created_at, used_at

### tasks
- id, user_id, title, scene, requirement_category, task_description, status, created_at, updated_at

### task_attachments
- id, task_id, filename, file_type, file_url, parsed_text, created_at

### task_dashboard
- id, task_id, screenshot_url, parsed_data (JSON), created_at

### model_outputs
- id, task_id, model_name, content_text, files (JSON), created_at

### chat_messages
- id, task_id, role (user/assistant), content, step_type, created_at

### evaluation_reports
- id, task_id, model_name, report_text, scores (JSON), created_at, updated_at

### admin_users
- id, username, password_hash, created_at
